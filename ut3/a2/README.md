# UT3-A2 Elige tu IDE ideal

Actividad diseñada para escenario telemático

## Objetivos
- Analizar y comparar distintos entornos de desarrollo.  
- Reflexionar sobre los criterios que influyen al elegir un IDE.  
- Justificar una elección técnica de forma razonada.



## Contexto
Imagina que formas parte de un nuevo equipo de desarrollo que va a iniciar un proyecto web o de aplicaciones.  
Antes de empezar, el equipo debe decidir **qué IDE utilizar**.  
Tú eres la persona encargada de **analizar varias opciones y justificar la elección final** ante el grupo.



## Tareas

### Investigación rápida (15 min)
Busca información sobre **tres IDE distintos** (pueden ser de lenguajes o entornos diferentes).  
Algunos ejemplos:
- Visual Studio Code  
- IntelliJ IDEA  
- Eclipse x 
- NetBeans  
- PyCharm x 
- Android Studio  
- Replit o Gitpod (IDE online)x



### Análisis comparativo (20 min)
Completa una tabla con la siguiente estructura:

| Criterio                                                                         | IDE 1: PyCharm                                                                          | IDE 2: Replit                                                                  | IDE 3: Eclipse                                                                        |
|:---------------------------------------------------------------------------------|:----------------------------------------------------------------------------------------|:-------------------------------------------------------------------------------|:--------------------------------------------------------------------------------------|
| Lenguajes principales que soporta                                                | Python (principalmente), también HTML, JS, SQL, etc.                                   | Varios lenguajes: Python, JavaScript, C++, Java, HTML, etc.                    | Java (principalmente), C++, Python, PHP, etc.                                         |
| Peso o consumo de recursos                                                       | Alto, requiere buena memoria y CPU.                                                     | Bajo, basado en la nube (poco consumo local).                                  | Medio-alto, depende de los complementos instalados.                                   |
| Facilidad de instalación                                                        | Fácil, instalación local mediante ejecutable.                                         | Muy fácil, solo requiere navegador web.                                       | Media, requiere instalación y configuración de JDK y plugins.                       |
| Funciones destacadas (depuración, refactorización, integración con Git, etc.) | Excelente depuración, autocompletado, refactorización, integración con Git, testing. | Ejecución instantánea en línea, colaboración en tiempo real, autoguardado. | Depuración avanzada, integración con Maven, Git, y herramientas de desarrollo Java. |
| Interfaz / usabilidad                                                            | Moderna y cómoda, intuitiva para Python.                                               | Muy sencilla e intuitiva, ideal para principiantes.                            | Interfaz más técnica y compleja, menos amigable para novatos.                       |
| Comunidad y soporte                                                              | Amplia comunidad y soporte de JetBrains.                                                | Comunidad activa en línea, soporte limitado en funciones avanzadas.           | Amplia comunidad open-source y soporte extenso para Java.                             |
| Licencia / coste                                                                 | Versión gratuita (Community) y de pago (Professional).                                 | Gratuito con opciones premium.                                                 | Gratuito (open-source).                                                               |


### Conclusión y justificación (10 min)
Redacta un texto breve (unas 10 líneas) donde respondas:

- ¿Cuál de los tres IDE recomendarías para tu grupo de trabajo?
- ¿Por qué crees que se adapta mejor a un proyecto de desarrollo web o de aplicaciones? 
- ¿Qué criterio has valorado más (rendimiento, comodidad, lenguaje, integración, etc.)?

Recomendaría PyCharm para nuestro grupo de trabajo, especialmente si el proyecto está enfocado en desarrollo con Python o aplicaciones web ligeras. 
Su entorno es potente, con un buen sistema de depuración, integración fluida con Git y herramientas de testing, lo que mejora la productividad del equipo. 
Aunque consume más recursos que Replit, su rendimiento y funciones profesionales compensan esta desventaja.
Considero que PyCharm se adapta mejor a un proyecto de desarrollo web o de aplicaciones por su facilidad para manejar frameworks y su entorno de trabajo ordenado.
El criterio que más he valorado es la integración y funcionalidad, ya que permiten optimizar el flujo de desarrollo y colaboración en equipo.
